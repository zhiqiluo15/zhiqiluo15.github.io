Git 协作强制规范 (AI 必须严格遵守)
#1. 禁止直接在 main 分支修改
任何改动（新功能、新章节、样式、bug 修复）必须先新建分支
分支命名：feature/xxx、fix/xxx、style/xxx、docs/xxx
示例：feature/添加第七章、fix/移动端菜单错位
main 分支只保留确认可展示的稳定版本
#2. 开工必须先同步最新代码
每次开始任务，第一条命令必须是：
bash
运行
git checkout main && git pull
#3. 代码读取与上下文强制约束（控制 Token 消耗，禁止全仓库扫描）
任何任务启动时，严禁自动读取项目根目录全仓库（#Workspace），不允许无差别遍历全部文件夹、依赖目录、静态资源、测试文档。
读取代码必须遵循最小范围原则：仅读取当前任务对应的业务模块文件夹，统一使用 #Folder 指定模块路径 限定读取范围，未手动指定文件夹时，仅读取当前打开编辑的单个文件。
永久屏蔽以下目录读取权限，任何场景都不得加载进上下文：
node_modules/、dist/、build/、.venv/、tests/、docs/、缓存目录、日志文件、legacy 废弃旧模块。
单次会话只处理单一模块需求；当前模块调试完成后，主动提示用户新建空白对话，截断长上下文重复计费，减少额度消耗。
若业务逻辑需要跨模块查阅代码，必须先弹窗文字询问用户是否允许扩大扫描范围，未得到用户确认前，禁止检索项目其他文件夹
