# 变更日志

> **修改代码前，请先阅读本日志。**  
> 这里记录了设计决策和重构意图，避免重复踩坑或推翻之前的合理选择。

## 模板

```markdown
## YYYY-MM-DD HH:mm

### 新增
- xxx

### 修改
- xxx

### 修复
- xxx

### 重构
- xxx

### 清理
- xxx
```

### 说明
- **日期**：按 `YYYY-MM-DD HH:mm` 格式记录，精确到分钟
- **分类**：每项变更归入对应分类（新增 / 修改 / 修复 / 重构 / 清理）
- **描述**：简要说明变更内容及原因，方便回溯
- **顺序**：最新的变更记录放在最前面
